## 1. Что такое VPS и зачем он нужен

**VPS (Virtual Private Server)** — это виртуальный сервер с выделенными ресурсами (CPU, RAM, диск), который работает как отдельный компьютер.

### Когда нужен VPS:

- 🌐 Запуск сайта или лендинга
- 🤖 Telegram / Discord / WhatsApp боты
- 🔌 API, backend, микросервисы
- 🧪 Тестовые и учебные проекты
- 🤖 AI-агенты, парсеры, воркеры

### Когда VPS НЕ нужен:

- 📄 Простая визитка без логики (дешёвый shared-хостинг)
- 😕 Если ты вообще не хочешь разбираться с сервером

## 4. SSH — подключение к серверу

### 4.1 Что такое SSH

**SSH (Secure Shell)** — это защищённый способ подключиться к серверу и управлять им через терминал.

**Аналогия:**  
SSH = удалённый рабочий стол, но только в текстовом формате.

### 4.2 Подключение с разных систем

#### **macOS / Linux**

У тебя уже есть терминал. Открываешь и пишешь:

```bash
ssh root@185.123.45.67
```

(замени `185.123.45.67` на IP твоего сервера из панели Timeweb)

Если первое подключение:
```
The authenticity of host can't be established.
Are you sure you want to continue? (yes/no)
```
Пишешь: `yes`

Потом вводишь пароль (если настроил через пароль).

✅ **Ты подключён!**

---

#### **Windows**

**Способ 1: PowerShell** (Windows 10+)

PowerShell уже поддерживает SSH:
```powershell
ssh root@185.123.45.67
```

**Способ 2: PuTTY** (если нет SSH)

1. Скачиваешь [PuTTY](https://www.putty.org/)
2. В поле **Host Name** вводишь IP: `185.123.45.67`
3. Port: `22`
4. Connection type: SSH
5. Нажимаешь **Open**
6. Вводишь `root` как логин
7. Вводишь пароль

**Способ 3: Консоль в браузере Timeweb** (самый простой)

1. Открываешь панель управления Timeweb
2. Выбираешь свой сервер
3. Нажимаешь кнопку **"Консоль"**
4. Открывается терминал прямо в браузере
5. Вводишь логин `root` и пароль

✅ **Удобно для новичков — не нужно ничего устанавливать!**

---

### 4.3 SSH Key (рекомендуемый способ)

**SSH Key** — это пара ключей (приватный + публичный), которые заменяют пароль.

**Плюсы:**
- Безопаснее пароля
- Не нужно каждый раз вводить пароль
- Защита от брутфорса

#### Генерация SSH ключа

**macOS / Linux:**
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Нажимаешь Enter → Enter → Enter (сохранится в `~/.ssh/id_ed25519`)

**Windows PowerShell:**
```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
```

#### Копирование публичного ключа на сервер

**Через интерфейс Timeweb (проще всего):**

1. При создании сервера выбираешь "SSH ключ"
2. Вставляешь содержимое файла `~/.ssh/id_ed25519.pub`
3. Сервер создаётся уже с твоим ключом

**Вручную (если сервер уже создан):**

1. Смотришь публичный ключ:
```bash
cat ~/.ssh/id_ed25519.pub
```

2. Копируешь вывод (начинается с `ssh-ed25519 ...`)

3. Подключаешься к серверу по паролю:
```bash
ssh root@185.123.45.67
```

4. На сервере создаёшь файл:
```bash
mkdir -p ~/.ssh
nano ~/.ssh/authorized_keys
```

5. Вставляешь скопированный публичный ключ
6. Сохраняешь: `Ctrl+O` → Enter → `Ctrl+X`
7. Права доступа:
```bash
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~/.ssh
```

Теперь можешь подключаться без пароля:
```bash
ssh root@185.123.45.67
```

## 6. Первоначальная настройка сервера

### 6.1 Обновление системы

Сразу после создания VPS:

```bash
sudo apt update && sudo apt upgrade -y
```

Это обновит все пакеты до последних версий.

⏱️ Займёт 2-5 минут в зависимости от количества обновлений.

### 6.2 Создание нового пользователя (безопасность)

❌ **Плохо:** работать всегда под `root`  
✅ **Хорошо:** создать отдельного пользователя

```bash
# Создать пользователя (например, deploy)
adduser deploy

# Система попросит ввести пароль и информацию
# Вводишь пароль 2 раза, остальное можно пропустить (Enter)

# Добавить в группу sudo (чтобы мог использовать sudo)
usermod -aG sudo deploy

# Переключиться на нового пользователя
su - deploy
```

Теперь подключайся через:
```bash
ssh deploy@185.123.45.67
```

💡 **Почему это важно:**
- Если взломают `root` — доступ ко всему серверу
- Если взломают `deploy` — ограниченный доступ, меньше урона

### 6.3 Настройка файрвола (UFW)

**UFW (Uncomplicated Firewall)** — простой файрвол для Ubuntu.

```bash
# Включить UFW
sudo ufw enable

# Разрешить SSH (порт 22)
sudo ufw allow 22

# Разрешить HTTP (порт 80)
sudo ufw allow 80

# Разрешить HTTPS (порт 443)
sudo ufw allow 443

# Проверить статус
sudo ufw status
```

Вывод:
```
Status: active

To                         Action      From
--                         ------      ----
22                         ALLOW       Anywhere
80                         ALLOW       Anywhere
443                        ALLOW       Anywhere
```

⚠️ **ВНИМАНИЕ:** Не забудь разрешить порт 22 (SSH) ПЕРЕД включением UFW! Иначе потеряешь доступ к серверу.

## 7. Как работать с сервером (workflow)

### 7.1 Типичный процесс деплоя

```
1. Разработка локально (на своём компьютере)
   ├─ Пишешь код
   └─ Тестируешь

2. Коммит в Git
   ├─ git add .
   ├─ git commit -m "Added landing page"
   └─ git push origin main

3. На сервере
   ├─ ssh deploy@server
   ├─ cd /var/www/my-project
   ├─ git pull origin main
   └─ Перезапуск сервисов (nginx, бот и т.д.)

4. Проверка
   └─ Открываешь сайт в браузере или проверяешь бота
```

### 7.2 Работа с логами

Логи — это текстовые файлы, где записывается что происходит.

```bash
# Логи nginx
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log

# Логи системы
sudo tail -f /var/log/syslog

# Логи PostgreSQL
sudo tail -f /var/log/postgresql/postgresql-16-main.log
```

💡 **tail -f** — показывает последние строки и обновляет в реальном времени (Ctrl+C чтобы выйти)

**Примеры что смотреть в логах:**

```bash
# Последние 50 строк логов nginx
sudo tail -50 /var/log/nginx/error.log

# Поиск ошибок
sudo grep "error" /var/log/nginx/error.log

# Логи за сегодня
sudo journalctl --since today
```

### 7.3 Автозапуск процессов (systemd)

Чтобы бот или сервер запускались автоматически после перезагрузки, используется **systemd**.

**Пример для Telegram бота:**

1. Создаёшь файл сервиса:
```bash
sudo nano /etc/systemd/system/telegram-bot.service
```

2. Содержимое:
```ini
[Unit]
Description=Telegram Bot
After=network.target

[Service]
Type=simple
User=deploy
WorkingDirectory=/var/www/telegram-bot
ExecStart=/usr/bin/python3 /var/www/telegram-bot/bot.py
Restart=always

[Install]
WantedBy=multi-user.target
```

3. Сохраняешь: `Ctrl+O` → Enter → `Ctrl+X`

4. Активируешь:
```bash
sudo systemctl daemon-reload
sudo systemctl enable telegram-bot
sudo systemctl start telegram-bot
```

5. Проверяешь статус:
```bash
sudo systemctl status telegram-bot
```

**Управление:**
```bash
sudo systemctl start telegram-bot    # Запустить
sudo systemctl stop telegram-bot     # Остановить
sudo systemctl restart telegram-bot  # Перезапустить
sudo systemctl status telegram-bot   # Статус
```

**Смотреть логи:**
```bash
sudo journalctl -u telegram-bot -f
```

## 8. Безопасность VPS

### 8.1 Основные правила

❌ **НИКОГДА не делай:**
- Работать всегда под `root`
- Хранить пароли в коде
- Открывать все порты
- Использовать слабые пароли
- Игнорировать обновления
- Отключать файрвол

✅ **ВСЕГДА делай:**
- Создавай отдельного пользователя
- Используй SSH ключи вместо паролей
- Настрой файрвол (UFW)
- Используй `.env` для секретов
- Обновляй систему регулярно
- Делай бэкапы

### 8.2 Отключение входа по паролю для root

После настройки SSH ключей:

```bash
sudo nano /etc/ssh/sshd_config
```

Найди и измени:
```
PermitRootLogin no
PasswordAuthentication no
```

Сохрани и перезапусти SSH:
```bash
sudo systemctl restart sshd
```

Теперь подключение только по SSH ключу для пользователя `deploy`.

### 8.3 Fail2Ban (защита от брутфорса)

**Fail2Ban** блокирует IP после нескольких неудачных попыток входа.

```bash
sudo apt install fail2ban -y

sudo systemctl enable fail2ban
sudo systemctl start fail2ban
```

Проверка:
```bash
sudo fail2ban-client status sshd
```

💡 Fail2Ban автоматически банит IP на 10 минут после 5 неудачных попыток входа.

### 8.4 Автоматические обновления

```bash
sudo apt install unattended-upgrades -y

sudo dpkg-reconfigure --priority=low unattended-upgrades
```

Выбери **Yes** — система будет автоматически устанавливать обновления безопасности.

### 8.5 Использование .env для секретов

❌ **ПЛОХО:**
```python
# bot.py
BOT_TOKEN = "123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"

```

✅ **ХОРОШО:**
```python

load_dotenv()
```

### 9.3 Бэкапы

❌ **Не полагайся только на сервер**  
✅ **Делай регулярные бэкапы**

**Способы:**

1. **Snapshots в панели Timeweb**
   - Заходишь в панель управления
   - Выбираешь сервер → "Снимки"
   - Создаёшь snapshot (занимает 2-5 минут)
   - Хранится 7 дней бесплатно

2. **Git для кода**
   - Весь код должен быть в Git (GitHub/GitLab)
   - `git push` после каждого изменения

3. **Экспорт БД PostgreSQL:**
```bash
# Создать дамп
pg_dump database_name > backup.sql

# Восстановить из дампа
psql database_name < backup.sql
```

💡 **Рекомендация:** делай бэкап перед каждым большим обновлением.

## 12. FAQ

Для большинства задач — **да**. Некоторые провайдеры включают IPv4 в тариф (VDSina, VDSka, RuVDS, Selectel, REG.Cloud), другие берут отдельно (Timeweb 180 ₽, Beget 55 ₽, Sprintbox ~190 ₽).

### Что с бэкапами?

- **Бесплатно**: Beget, VDSina (еженедельные)
- **Платно**: Timeweb (6 ₽/ГБ), VDSka (~50–100 ₽), Selectel (в промо 0 ₽)
